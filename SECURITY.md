# Security Policy

We take the security of this CI/CD repository seriously.
This document explains how to report security issues related to pipelines and automation.

## Scope
This repository manages automated build, test, and delivery workflows.
Security risks related to workflow execution and access are in scope.

## Reporting a Security Issue
Do not report CI/CD security issues in public issues or comments.
Report them privately to the CI/CD maintainers.
Include details about the affected workflow and possible impact.

## What to Include
- A clear description of the issue
- The workflow or stage affected
- Any error messages or logs if available

## Response Process
The team will review the report promptly.
High-risk issues will be fixed with priority.
You will be notified once the issue is resolved.

Thank you for helping keep our CI/CD processes secure.
